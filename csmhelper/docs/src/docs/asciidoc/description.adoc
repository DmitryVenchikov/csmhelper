== Описание сервиса

`csmhelper` — это веб-приложение на ASP.NET Core MVC (`net8.0`), вспомогательный инструмент CSM/CTL для работы с корпоративным экземпляром Jira (`https://jira.moscow.alfaintra.net`). Сервис не имеет собственной базы данных — все доменные данные он берёт из Jira REST API. Локально на диск пишутся только лог-файлы аутентификации и доступа в каталог `logs/`.

. Сервис предоставляет возможность:

.. Аутентифицироваться в Jira по логину/паролю и хранить эти учётные данные в `Session` ASP.NET Core (`JiraUsername`, `JiraPassword`, `IsAuthenticated`)

.. Создавать связку из трёх связанных задач (Аналитика → Разработка → Тестирование) в выбранном проекте Jira с автоматическими `Blocks`-связями

.. Массово обрабатывать задачи Jira (поиск по JQL-фильтру с последующим удалением или переводом в целевой статус) через раздел *Cleanup*

.. Строить диаграмму Ганта по задачам Jira: тянуть открытые задачи проектов, парсить тип (`[TEST]`, `[DEV BACK]`, `[DEV FRONT]`, `[SA]`) и Story Points, распределять по сотрудникам с учётом рабочего календаря, обедов, выходных и блокирующих связей, считать критический путь и утилизацию

.. Логировать на диск успешные/неуспешные попытки входа, статистику пользователей по дням и месяцам, а также посещения вспомогательных страниц (`Timer`)
