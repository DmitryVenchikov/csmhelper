=== `GET /auth/stats` Статистика входов и неудачных попыток


==== Назначение
Сервисный (мониторинговый) метод. Возвращает агрегированную статистику входов за текущий день и месяц, а также сводку неудачных попыток. Данные читаются из текстовых файлов в каталоге `logs/`, которые ведутся методом `POST /Auth/Login`.

Метод не требует аутентификации (исторически опубликован как `[Route("auth/stats")]`).

==== Алгоритм работы
. Система получает запрос `GET /auth/stats`
. Система вычисляет пути к файлам статистики:
.. `logs/daily_stats_<yyyy-MM-dd>.txt` — дневная статистика входов
.. `logs/monthly_stats_<yyyy-MM>.txt` — месячная статистика входов
.. `logs/failed_stats_<yyyy-MM-dd>.txt` — статистика неудачных попыток за день
. Система читает каждый из файлов:
.. Если файл существует — содержимое читается построчно (`File.ReadAllLines`)
.. Если файл отсутствует — массив строк пуст
. Система суммирует общее число неудачных попыток за день, парся каждую строку `username | attempts | last | reason` и складывая поле `attempts`
. Система формирует JSON-ответ:
.. `Date` — текущая `DateTime.Now`
.. `DailyStats` — массив строк дневной статистики
.. `MonthlyStats` — массив строк месячной статистики
.. `FailedStats` — массив строк по неудачным попыткам
.. `TotalFailedToday` — суммарное число неудачных попыток за день
. Если в обработке возникло исключение — Система пишет `LogError` и возвращает JSON `{ error = <ex.Message> }`
. Система возвращает ответ с HTTP-кодом = 200 OK
